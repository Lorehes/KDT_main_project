# 프로젝트 전체 진척도 (2026-06-05 기준)

> 통합기획서 Phase 1~4 로드맵 대비 현재 어디까지 왔고, 무엇이 남았는지 정리.
> 직전 세션 상세 흐름은 [[2026-06-04_세션보고_M1_Stage2_LLM]].

## 0. 한 장 요약

- **Phase 1 MVP 기준 약 30% 완료** / Phase 1~4 전체로는 약 10~15%
- 데이터 토대(수집·룰분류) ✅ + Stage 2 LLM 분류 ✅ 완성
- 핵심 가치 명제 충족엔 **Stage 3 RAG + 사용자/포트폴리오 + 알림 + 프론트엔드** 4영역이 남음
- 순수 개발 작업 합 ≈ **4~6주**, 외부 의존(카카오 검수/법률) 병행 시 **MVP 출시까지 약 6~10주**
- 권고 다음 액션: **(A) Stage 3 RAG Spec 작성 + (B) M2 User/Auth 병행 구현**

## 5. 사용자 가치 명제 충족도

> 통합기획서 §1: "보유 종목 DART 공시를 실시간으로 받아 호재/악재 의미를 자연어로 해석"

| 단계 | 상태 | 비고 |
|------|------|------|
| ① DART 공시 실시간 수집 | ✅ 1분 폴링 가동 | |
| ② 보유 종목 매칭 | ❌ User/Portfolios 도메인 없음 | M2 필요 |
| ③ 호재/악재 분류 | 🟡 Stage 2 가동, 단 정확도 한계 | 500건 측정: 99% NEUTRAL, POSITIVE 0건. Stage 3 보강 필수 |
| ④ 자연어 해석 (summary 생성) | 🟡 가동, 단 가치 낮음 | "TREASURY_STOCK 분류를 기재" 같은 영어 enum 노출. Stage 3+ 정확도 보강 필요 |
| ⑤ 알림 발송 (카카오 알림톡) | ❌ Notification 도메인 + 비즈 검수 둘 다 없음 | M3 필요 |

→ **현재 사용자가 체감할 수 있는 가치는 0**. 모든 출력은 백오피스/관리자 관점.

## 8. 우선순위 매트릭스 (가치 × 작업량)

```
              ↑ 가치 높음
              │
       Stage 3 RAG       │   Stage 4 LLM 2차
       (모델 정확도 본질) │   (Stage 3 결과 결합)
       1~2주             │   3~5일
       ────────────────  │  ─────────────────
       User/Auth         │   Notification
       (전제 조건)       │   (사용자 도메인 의존)
       3~5일             │   5~7일
              │
              ↓ 가치 낮음
       작업 큼  ───────────── 작업 작음
```

### 권고 진행 순서

| 순서 | 항목 | 이유 |
|------|------|------|
| **1차 (지금)** | **Stage 3 RAG Spec 작성**(`/dc-plan`) | 모델 한계 본질 해결책. 큰 작업이라 설계 단계만이라도 미리 시작 |
| **1차 병행** | **M2 User/Auth + Portfolios 구현** | Stage 3와 의존 없음, 2명 이상이면 분담 가능 |
| **2차** | Stage 3 RAG 구현 | Spec 확정 후 wave 단위 진입 |
| **2차** | Stage 4 LLM 2차 (Stage 3 끝나면 즉시) | 호재/악재 정확도 보강의 마지막 조각 |
| **3차** | M3 Notification 도메인 | User/Portfolios 끝난 뒤 시작 가능 |
| **3차 병행** | 카카오 알림톡 비즈 검수 (행정 진행) | 검수 통과까지 2~4주 소요라 미리 시작 |
| **4차** | Frontend 핵심 페이지 (로그인 → 대시보드 → 포트폴리오 → 공시) | User/Auth 백엔드 끝난 뒤 |
| **4차 병행** | 법률 검토 + 베타 테스터 모집 | 외부 의존 |
| **5차** | CI/CD + 모니터링 + 배포 인프라 | MVP 출시 직전 |
