## 3. 만들어진 것 — Stage 2 LLM 분류 파이프라인

### 3.1 파이프라인 한 줄 요약

> DART OpenAPI에서 1분마다 신규 공시 폴링 → Stage 1 룰 분류 → **(NEW) 이벤트 발행 → Stage 2 LLM이 호재/중립/악재로 분류 → 결과를 analysis_results에 저장 → 알림 도메인용 이벤트 발행** → 사용자가 REST로 조회

### 3.2 구성 요소 (신규)

| 컴포넌트 | 역할 |
|----------|------|
| `OllamaLlmClient` | Ollama HTTP API 호출 (qwen3:4b, think=false 필수) |
| `Stage2PromptBuilder` | 프롬프트 생성 + 자본시장법 가드 명시 |
| `PromptGuard` | 응답 후처리 — 9개 금지 키워드 + 신뢰도 임계 + 240자 cap |
| `Stage2Analyzer` | 분석 1건 처리 — UPSERT 멱등 + 파싱 재시도 1회 + silent fail |
| `AnalysisOrchestrator` | 이벤트 리스너 — 신규 공시 자동 분석 트리거 |
| `AnalysisBackfillService` | 미분석 공시 배치 처리 (워터마크 진행) |
| `AnalysisBackfillController` | `/admin/analysis/backfill/jobs` POST/GET |
| `analysis_jobs` 테이블(V13) | 배치 잡 상태/진행률 추적 |

### 3.3 동시성 설계

- `analysisExecutor` (코어 2/맥스 4) — 폴링 트리거 분석, 우선순위 높음
- `analysisBackfillExecutor` (코어 1/맥스 2) — 백필, 낮은 우선순위
- 격리 이유: Ollama RPS 한계. 백필이 폴링 큐를 막아 신규 공시 SLO 30초 위반하지 않도록.

### 3.4 환각/법적 가드 (3중 방어)

| 계층 | 방어 |
|------|------|
| **L1 프롬프트** | "정보 제공 목적, 자문 아님" 명시 + 금지 표현 prohibit |
| **L2 응답 후처리** | PromptGuard — 9개 금지 키워드 매칭 시 withheld=true 강제, 신뢰도 임계 0.6 미만 보류, 240자 cap |
| **L3 응답 직렬화** | `AnalysisResponse` DTO에 `disclaimer` + `reportInaccuracyPath` 강제 |

## 5. 운영 사고 3건 — 상세

### 5.1 사고 #1 — Postgres 데이터 통째 손실 (2026-06-04 오전)

**증상**: `docker compose up -d` 후 `disclosures` 테이블이 존재하지 않음.

**조사**:
- Docker named volume(`kdt_mainpj_dartcommons_pgdata`)이 **방금 생성된** 것으로 확인
- 이전 세션(2026-06-03)에 91,965건 적재한 볼륨이 6/3~6/4 사이에 통째 삭제됨
- Docker Desktop의 "Clean / Purge data" 또는 "Reset to factory defaults" 클릭 추정

**대응 (3중 방어 구축)**:
1. **bind mount 전환** — `~/data/dartcommons-pg` (Docker 관리 외부 디렉토리, Docker reset 무관)
2. **자동 백업** — `scripts/data_protection/backup_db.sh` (pg_dump|gzip, crontab 일 1회 03:00)
3. **레포 내 미러** — `Data/*.sql.gz` (`gitignore`의 `data/` 규칙이 case-insensitive 자동 보호)
4. **운영 가이드** — `docs/개발명세서/data_protection.md` (시나리오 A/B/C 복원 절차 + 절대 금지 명령)

**현재 상태**: 91,989건 보호 — bind mount + cron + 레포 미러 + 옛 named volume(fallback 보존).

### 5.2 사고 #2 — DartClient 페이지네이션 무한 루프 (2026-06-04 오후)

**증상**: 3년치 백필이 청크 12(`2026-02-18~2026-05-18`)에서 `pageNo=10,194` 도달, 7시간 stuck.

**원인**: `DartListResponse` DTO에 `total_page` 필드 미매핑 + `DartClient.fetchList`가 종료 조건을 `items.size() < PAGE_SIZE`에만 의존. DART API가 마지막 페이지 데이터를 반복 반환하는 케이스에서 무한 루프.

**핫픽스**:
- DTO에 `total_page` 추가
- 종료 조건 3중 가드: `total_page` 도달 → `items.size() < PAGE_SIZE` → `MAX_PAGES=3000` 절대 상한
- 단위 테스트 4건(`DartClientPaginationTest`): 정상 종료 / total_page 누락 / MAX_PAGES 가드 / status=013

**결과**: 91,989건 백필 정상 완료(이전 메모리 91,965건과 거의 동일).

### 5.3 사고 #3 — Analysis 백필 무한 루프 + LLM 미호출 (2026-06-05 새벽)

**증상**: 샘플 500건 백필 시작 직후 `chunks=427/10`(초과) + `analyzed=0` + `failed=21,350`.

**원인 두 가지**:
1. `OllamaLlmClient`의 `@Retryable maxAttemptsExpression = "#{@llmProperties.maxRetries()+1}"`이 SpEL 빈 참조 실패 (Spring Boot `@ConfigurationProperties` 빈명 규칙 차이) → 모든 호출이 LLM 도달 전 즉시 예외
2. `AnalysisBackfillService`의 `while(true)`이 같은 50건을 반복 조회 (실패한 disclosure는 `analysis_results`에 안 써져서 다음 iteration에도 같은 ids 반환)

**핫픽스**:
- `maxAttempts=3` 상수로 고정 (Spec `LLM_MAX_RETRIES=2`와 정합)
- **워터마크 진행** — 다음 청크는 직전 청크의 마지막 id+1부터 조회
- **safetyCap** — 예상 청크 수의 2배 상한
- **조기 중단 안전망** — 50건 시도 후 `analyzed=0`이면 즉시 throw (잘못된 환경 설정 즉시 노출)

**결과**: 재시도 잡 정상 완주 (analyzed=500/failed=0/36분39초).

## 6. 샘플 500건 실측 결과 (Wave 5)

### 6.1 성능

| 지표 | 값 | 목표 (기획서 §6.3) |
|------|----|-------|
| 평균 응답 시간 | **4.39초/건** | 3~15초 ✅ |
| p50 / p95 | 4.45 / 4.96초 | — |
| 처리량 | 13.7건/분 (1 worker 활성) | — |

### 6.2 분류 분포

| Sentiment | 건수 | 비율 |
|-----------|------|------|
| NEUTRAL | 495 | 99.0% |
| NEGATIVE | 5 | 1.0% |
| **POSITIVE** | **0** | **0.0%** |

| 지표 | 값 | 목표 |
|------|----|------|
| withheld 비율 | 0.8% (4건) | Spec G4 ≤30% ✅ |
| confidence p50 | 0.700 | — |
| confidence 분산 | 거의 없음 (0.70~0.85) | — |

### 6.3 핵심 발견

**✅ 잘 되는 것**:
- 명백한 악재 5건(생산중단/EMA 허가 철회/해산 사유 발생/소송 판결) 모두 정확하게 NEGATIVE로 인식
- 잘못된 호재 단정 0건 — 자본시장법 표현 위반 위험 0
- withheld 비율 매우 낮음(0.8%) → 사용자에게 "판단 보류" 노출 적음 → UX 양호

**⚠️ 우려되는 것**:
- POSITIVE 단정 0건 — 호재성 공시(자기주식취득 결과, 무상증자 등)도 모두 NEUTRAL → 모델이 사실상 "무지/회피" 패턴
- confidence가 0.7 근처에 수렴 — 모델 자기 확신 캘리브레이션 한계, 임계 기반 보류 가드의 실효성 약함
- PromptGuard "매수" 키워드 false positive 4건 발견 — "매수 권유"(자본시장법 위반)와 "매수 보고서"(사실 진술) 미구분

**Stage 2 단독으로는 "공시를 자연어로 해석"이라는 가치 명제(통합기획서 §1) 충족 불가**. Stage 3 RAG(과거 유사 공시 + KRX 5일 반응 결합) + Stage 4 LLM 2차 분석이 필수.
