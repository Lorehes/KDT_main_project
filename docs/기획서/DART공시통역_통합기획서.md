# DART 공시 통역 — 통합 기획서

> **개인 투자자를 위한 AI 기반 공시 해석 & 실시간 알림 플랫폼**
> **작성일**: 2026-05-27

---

## 1. 서비스 개요

### 1.1 한 줄 정의

> 보유 종목의 DART 공시를 실시간으로 받아 호재/악재 의미를 자연어로 해석해주는 개인 투자자용 알림 서비스.

### 1.2 해결하려는 문제

개인 투자자는 보유 종목의 DART 공시를 수신해도 다음 어려움에 직면한다.

- 공시 내용이 호재인지 악재인지 즉각 판단하기 어려움
- 전문 용어와 복잡한 서식으로 인한 정보 접근 장벽
- 시장 개장 중 실시간으로 공시를 모니터링하는 것이 물리적으로 불가능
- 정보 비대칭으로 인해 기관·외국인 대비 불리한 의사결정 환경

### 1.3 핵심 가치 사슬

| ① 수집 | ② 분류 | ③ AI 해석 | ④ 개인화 | ⑤ 전달 |
|--------|--------|---------|--------|--------|
| DART OpenAPI 실시간 폴링 | 공시 유형 분류 / 종목 매핑 | LLM 호재/악재 판단 + 요약 | 보유 종목 필터링 / 중요도 스코어링 | 카카오 알림톡 / 웹 대시보드 |

### 1.5 시장 크기

#### 전체 시장 규모

| 지표 | 수치 | 출처 |
|------|------|------|
| 주식 활동계좌 | 약 7,000만 개 | 한국예탁결제원 |
| 2025년 신규 계좌 | 867만 개 증가 | 금융투자협회 |
| ETF 투자자 수 | 약 600만+ | 금융투자협회 |

#### 도달 가능 시장 (TAM/SAM/SOM)

| 구분 | 정의 | 추정 규모 | 산정 근거 |
|------|------|--------|---------|
| TAM | 모든 주식 활동 계좌 | 7,000만 | 한국예탁결제원 |
| SAM | 월 1회 이상 거래 적극 투자자 | 약 800~1,200만 | 금융투자협회 추산 |
| SOM | 페르소나 A~D 해당층 | 약 200~300만 | SAM의 25~30% |

> **주의**: 본 시장 규모는 공식 통계와 산업 추산 근거이며, 실제 도달 사용자는 마케팅 채널과 제품 완성도에 따라 달라질 수 있다.

## 2. 핵심 가치 제안

### 2.1 차별화 포인트 4가지

1. **공시 원문 → 자연어 요약**: "전환사채 1,000억 발행 결정"을 "주식 추가 발행 가능성이 있어 기존 주주에게 불리할 수 있음"으로 변환
2. **과거 유사 공시 패턴 학습**: "이 회사의 동일 유형 공시 시 평균 주가 -6.3% 반응"
3. **개인 보유 종목 맥락 반영**: "평균 매수가 5만원, 현재가 손익선 위협 가능"
4. **하이브리드 분석**: 룰 기반 정확성 + RAG 보강으로 안정적 분석

### 2.2 기존 서비스와의 차이

#### (1) DART 공식 모바일 앱

| 항목 | DART 공식 앱 | 본 서비스 |
|------|----------|---------|
| 알림 내용 | 공시 제목만 푸시 | 호재/악재 판단 + 3줄 요약 |
| 의미 해석 | 없음 | LLM 자연어 해석 |
| 과거 패턴 | 없음 | 동일 회사 유사 공시 평균 주가 반응 |
| 보유 종목 맥락 | 단순 관심 등록 | 매수가 입력 시 손익 영향 분석 |

#### (2) 증권사 앱 (토스/카카오페이/키움)

| 항목 | 증권사 앱 | 본 서비스 |
|------|--------|---------|
| 핵심 기능 | 거래 (매수/매도) | 공시 분석 전문 |
| 공시 깊이 | 단순 표시, 부가 기능 | 메인 가치 |
| 객관성 | 자사 거래 유도 가능성 | 중립적 분석 |
| 서비스 집중도 | 낮음 (종합 플랫폼) | 높음 (공시 특화) |

#### (3) 금융 뉴스 앱 / 종목 토론방

| 항목 | 금융 뉴스 / 토론방 | 본 서비스 |
|------|------------|---------|
| 정보 출처 | 기자 / 일반 투자자 | DART 공식 원본 |
| 시차 | 수십 분 ~ 수 시간 | 30초 이내 |
| 정확성 | 가변 (루머 포함) | 공식 공시 기반 |
| 보도 범위 | 큰 이슈만 | 모든 공시 |

#### (4) 경쟁사 비교 요약

| 비교 항목 | 네이버·카카오페이 증권 | 본 서비스 |
|---------|----------------|---------|
| 공시 자연어 해석 | ✗ | ✓ |
| 호재/악재 판단 | ✗ | ✓ |
| 개인 포트폴리오 연동 | 부분 | ✓ |
| 커스텀 알림 설정 | ✗ | ✓ |
| 서비스 집중도 | 낮음 (종합) | 높음 (공시 특화) |

### 2.3 글로벌 벤치마킹

본 서비스는 다음 3가지 글로벌 사례를 분석하여 설계되었다.

#### Robinhood Gold (구독 멤버십 모델)

- Gold 멤버 350만 (+76% YoY), 2024 매출 $2.95B, 순이익 $1.41B
- 무료 계좌 유지 + Gold 구독 시 고급 리서치·마진거래·금리 혜택 묶음
- 단계적 기능 개방으로 락인 효과 (Free → Gold → Institutional)
- **본 서비스 적용**: 공시 알림은 무료, 과거 패턴 분석·호재악재 상세 근거는 유료 구조

#### Seeking Alpha (퀀트 레이팅 + 알림)

- MAU 2,000만+, Quant Strong Buy 2024년 수익률 +37.15% (S&P 500: +12.75%)
- 100+ 지표 자동 분석, 보유 종목 등급 변동 시 즉시 알림
- 수치로 검증된 성과가 유료 전환의 핵심 근거
- **본 서비스 적용**: 공시 영향도 점수(1~10) 수치화로 Pro 가치 증명

#### Fiscal.ai (B2C → B2B API 확장)

- 등록 유저 35만+, 기관 API 고객 70개+, Series A $10M
- LLM + RAG로 SEC 공시·실적 발표 2.4M건 AI 처리, VanEck 등 기관 확보
- 리테일 바이럴 → B2B 스케일 공식
- **본 서비스 적용**: B2C 커뮤니티 확보 → 증권사·핀테크 B2B API 확장

## 3. 서비스 범위

### 3.1 종목 커버리지

| 범위 | 종목 수 | 정책 |
|------|------|------|
| 코스피 200 + 코스닥 150 | 350종목 | 자동 분석 |
| 추가 등록 요청 | 사용자 요청 가능 | 7일 이내 검토 후 추가 |

**선정 근거**: 코스피 200 + 코스닥 150은 한국 증시 거래대금의 약 80%를 커버한다 (한국거래소 통계 기반).

**미커버 영역**:

| 미커버 항목 | 이유 |
|---------|------|
| 우선주 | 보통주와 공시 동일, 분리 가치 낮음 |
| 스팩 (SPAC) | 합병 전 비활성, 공시 거의 없음 |
| 채권 | 공시 체계 다름, 별도 모델 필요 |
| 상장폐지 위험 종목 | 정보 부정확 가능성, 법적 리스크 |
| 해외 주식 | DART는 한국 공시만, 별도 소스 필요 (장기 확장) |

### 3.2 공시 유형 커버리지

**모든 공시 (DART 80여 종)** 분석. 분석 깊이는 BM 등급별 차등.

### 3.3 분석 깊이 단계

```
[Stage 1: 호재/중립/악재 분류]      ← Free
        ↓
[Stage 2: 3줄 자연어 요약]          ← Free
        ↓
[Stage 3: RAG 유사 공시 검색]       ← Pro
        ↓
[Stage 4: 과거 주가 반응 분석]      ← Pro
        ↓
[Stage 5: 회사 재무 + 업황 분석]    ← Premium
```

## 4. 서비스 흐름

> **변동 가능성 안내**: 본 섹션의 흐름은 초기 설계안이다. 사용자 테스트, 기술 제약, 카카오 알림톡 정책 변경 등에 따라 조정될 수 있다.

### 4.1 회원가입 흐름

```
[1] 랜딩 페이지 진입
       ↓
[2] "시작하기" 클릭
       ↓
[3] 가입 방식 선택
    - 이메일 회원가입
    - 카카오 로그인
    - 구글 로그인 (2순위)
    - 네이버 로그인 (2순위)
       ↓
[4] 이메일 인증 / 토큰 검증
       ↓
[5] 약관 동의 (서비스 약관 + 개인정보 처리방침)
       ↓
[6] 휴대폰 인증 (선택, 카카오 알림톡 사용 시 필수)
       ↓
[7] 닉네임 + 투자 경험 입력 (선택)
       ↓
[8] 가입 완료 → 대시보드
```

### 4.2 보유 종목 등록 흐름

```
[1] "종목 등록" 클릭
       ↓
[2] 입력 방식 선택
    ┌─ 수동 검색 + 추가
    │   - 종목명/코드 검색 (자동완성)
    │   - 선택 후 매수가/수량 입력 (선택)
    │
    └─ 거래내역 CSV 업로드
        - 증권사 거래내역 다운로드 가이드
        - CSV 업로드 → 자동 추출 → 확인 → 저장
       ↓
[3] 등록 완료 (Free 3개 / Pro 무제한)
       ↓
[4] 커버리지 밖 종목이면 "추가 등록 요청" 안내
```

### 4.3 알림 설정 흐름

```
[1] "알림 설정" 진입
       ↓
[2] 알림 채널 선택
    1순위: 카카오 알림톡 (휴대폰 인증 + 채널 추가)
       ↓ (카카오 사용 불가 시)
    사용자가 텔레그램 또는 이메일 선택
       ↓
[3] 알림 빈도 설정
    - 즉시 푸시
    - 일 1회 (오전 9시)
    - 일 2회 (9시 + 18시)
    - 주 1회 (월요일)
       ↓
[4] 알림 종류 필터 (호재만 / 악재만 / 모두)
       ↓
[5] 거래시간 외 알림 정책
       ↓
[6] 설정 저장
```

### 4.4 공시 알림 발송 흐름 (시스템 자동)

```
[1] DART OpenAPI 1분 폴링 → 신규 공시 감지
       ↓
[2] 공시 메타데이터 추출 (Stage 1)
       ↓
[3] 커버 종목 여부 확인 (미커버 시 스킵)
       ↓
[4] 영향 받는 사용자 조회
       ↓
[5] LLM 분석 (Stage 2~5, BM 등급별)
       ↓
[6] 사용자별 알림 큐 적재 (즉시 / 다이제스트)
       ↓
[7] 채널별 발송 (카카오 / 텔레그램 / 이메일)
       ↓
[8] 발송 결과 기록 + 재시도 (최대 3회)
       ↓
[9] 사용자 클릭 → 웹 상세 화면
```

### 4.5 상세 분석 화면 흐름 (웹)

```
[1] 알림 링크 클릭
```

## 6. 하이브리드 RAG 상세 구현

> **변동 가능성 안내**: 본 섹션의 단계 구분과 처리 방식은 초기 설계안이다. 모델 선택, RAG 품질 검증 결과에 따라 단계 통합/분리 또는 순서 변경 가능성이 있다.

### 6.1 단계별 처리 흐름

#### Stage 1: 룰 기반 핵심 추출

```
[입력] DART OpenAPI 응답 (XML/JSON)
[처리]
1. 응답 파싱
2. 메타 필드 추출 (접수번호, 회사코드, 회사명, 제목, 제출일자)
3. 공시 유형 분류 (정규식 + 사전 룰, 80여 종)
4. 첨부 문서 다운로드 + 본문 텍스트 추출
[출력] 메타데이터 객체 + 공시 본문
[LLM 호출] 0회 / [비용] 0원
```

#### Stage 2: LLM 1차 분석 (분류 + 요약)

```
[입력] Stage 1 결과
[처리]
1. 프롬프트 구성 (분류 + 3줄 요약 요청)
2. LLM 호출 (LangChain4j → Ollama 또는 Cloud)
3. JSON 응답 파싱
[출력] 분류 (호재/중립/악재) + 신뢰도 + 3줄 요약
[LLM 호출] 1회 / [토큰] 입력 ~1,500 / 출력 ~200
```

#### Stage 3: 임베딩 + RAG 검색

```
[입력] 공시 본문
[처리]
1. 본문 임베딩 생성
2. Chroma 유사 공시 검색
   - 동일 회사 + 동일 유형 (최대 5건)
   - 동일 유형 다른 회사 (최대 5건)
3. 검색 결과에 과거 5일 주가 반응 결합 (KRX 데이터)
[출력] 유사 공시 리스트 (최대 10건, 각 5일 주가 반응 포함)
[LLM 호출] 임베딩 1회
```

#### Stage 4: LLM 2차 분석 (최종 판단)

```
[입력] Stage 2 결과 + Stage 3 검색 결과
[처리]
1. 프롬프트 구성 (1차 분석 + 과거 사례 종합)
2. LLM 호출
3. 응답 파싱
[출력] 최종 분류 + 예상 주가 반응 + 근거 + 신뢰도
[LLM 호출] 1회 / [토큰] 입력 ~2,000 / 출력 ~300
```

#### Stage 5: Premium - 재무/업황 분석

```
[입력] Stage 4 결과 + 회사 코드
[처리]
1. 회사 분기 재무 데이터 조회 (DART 정기보고서)
2. 업황 데이터 조회 (공공 API)
3. LLM 프롬프트 구성 (재무 + 업황 영향 분석)
4. LLM 호출
[출력] 재무 영향 + 업황 맥락 + 리스크 평가
[LLM 호출] 1회 / [토큰] 입력 ~3,000 / 출력 ~500
```

### 7.2 라벨링 데이터 구축 전략

LLM 분류기의 정확도 검증과 프롬프트 튜닝을 위한 호재/악재 라벨링 데이터가 필요하다. 3단계로 진행한다.

#### 1순위: 공개 데이터셋 탐색

| 데이터셋 | 출처 |
|--------|------|
| AI 공시 평가 데이터셋 | 금융 빅데이터 플랫폼 |
| KRX 시장정보 데이터 | 한국거래소 |
| AI Hub 금융 자연어 데이터 | NIA |

검증: 라벨링 기준 확인 → 샘플 100건 검토 → 본 서비스 정의와 일치 시 즉시 활용.

#### 2순위: LLM 셀프 라벨링 + 검수

```
[1] 과거 3년치 공시 중 1,000~2,000건 추출
[2] Cloud LLM으로 자동 라벨링 (호재/중립/악재 + 이유)
[3] 팀원 2~3명이 무작위 100건 독립 검수
[4] LLM 라벨과 일치율 측정
    - 90% 이상 → 채택
    - 80~90% → 프롬프트 개선 후 재시도
    - 80% 미만 → 3순위로 이동
[5] 학습 80% / 검증 20% 분리
```

#### 3순위: 팀원 직접 라벨링

```
[1] 라벨링 가이드라인 작성 (경계 사례 예시 포함)
[2] 팀원별 할당 (5명 × 200건 = 1,000건, 또는 100건만)
[3] Jupyter Notebook + ipywidgets UI로 라벨링
[4] 100건 교차 검증 (Cohen's Kappa) → 불일치 토론 후 합의
[5] 최종 데이터셋 확정
```

#### 라벨링 데이터 활용

```
구축 데이터 → 학습/검증 분리 → Few-shot 프롬프트 구성
→ 정확도 측정 (Precision/Recall/F1) → KPI "정확도 90%" 검증
```

- 재무 데이터: 분기 1회 배치
- 종목 기본 정보: 분기 1회 검증

## 11. 법적/규제 고려사항

### 11.1 주요 리스크 및 회피 방법

#### 리스크 1: 자본시장법상 투자 자문업 경계

자본시장법 제6조에 의해 투자 자문업은 금융위원회 등록 필수. "추천", "매수/매도 권유" 표현 사용 시 경계 침범 가능.

| 회피 방법 | 내용 |
|---------|------|
| 표현 가이드라인 | 금지: "매수 추천", "꼭 사세요" / 권장: "참고하세요", "정보 제공". LLM 프롬프트에 명시 |
| 서비스 포지셔닝 | 메인 페이지 "정보 제공 도구" 명시, "투자 자문 아님" 안내 |
| 면책 조항 | 회원가입 시 동의 ("정보 제공이며 자문 아님", "책임은 사용자") + 동의 기록 저장 |
| 법무 검토 | 베타 출시 전 변호사 자문 (1회), 자문 반영하여 표현/약관 수정 |

#### 리스크 2: LLM 환각으로 인한 손실 책임

LLM이 잘못된 분석 제공 → 사용자 거래 → 손실 → 책임 분쟁.

| 회피 방법 | 내용 |
|---------|------|
| 하이브리드 구조 | 룰 기반 핵심 추출로 정확성 보장, LLM은 보조 역할로 한정 |
| 사용자 신고 | 모든 분석에 "부정확함" 신고 버튼, 다수 신고 시 비공개 처리 |
| 신뢰도 점수 공개 | LLM 산출 신뢰도(0~1) 표시, 확신도 낮으면 "판단 보류" 표시 |
| 면책 조항 강화 | "AI 분석은 참고용, 부정확 가능" 모든 알림 하단 명시 |
| 책임 한계 | 이용 약관에 손해배상 한도 명시 |

#### 리스크 3: 개인정보 (관심 종목, 매수가)

보유 종목과 매수가는 개인 자산 정보. 유출 시 사회적 손해.

| 회피 방법 | 내용 |
|---------|------|
| 암호화 저장 | 매수가/수량 AES-256 암호화, 키는 별도 KMS |
| 최소 수집 | 매수가/수량은 선택 사항, 휴대폰은 알림톡 사용 시에만 |
| 처리방침 | 수집 항목/목적/보유 기간 명시, 가입 시 동의 (선택/필수 분리) |
| 법규 준수 | 개인정보보호법/GDPR 준수, 데이터 조회/삭제권 보장 |
| 사고 대응 | 정기 보안 점검, 침해 시 72시간 내 통지 |

#### 리스크 4: 카카오 알림톡 정책 (2026.1.1 개편)

2026년 1월 1일부터 정책 개편, 정보성 메시지 기준 엄격화.

| 회피 방법 | 내용 |
|---------|------|
| 메시지 유형 분리 | 정보성(공시 알림)은 알림톡, 광고성(Pro 안내)은 친구톡/별도 채널 |
| 템플릿 사전 승인 | 모든 템플릿 카카오 사전 승인, 정책 변경 시 즉시 수정 |
| 폴백 채널 준비 | 승인 거부 시 텔레그램/이메일 즉시 전환 |
| 정기 모니터링 | 분기 1회 정책 변경 사항 확인 |

### 11.2 면책 조항 예시

> 본 서비스는 정보 제공을 목적으로 하며, 투자 자문이나 권유가 아닙니다. AI 분석 결과는 참고용이며 부정확할 수 있습니다. 모든 투자 결정의 책임은 사용자에게 있으며, 본 서비스는 투자 손실에 대한 책임을 지지 않습니다.

### 11.3 DART OpenAPI 한도 및 대응

DART OpenAPI는 호출 한도가 존재하며, 본 서비스 운영 시 이를 반드시 준수해야 한다.

#### 한도 정책 (공식 안내)

| 상시 운영 | 신규 공시 원문 다운로드 | 약 50회/일 |

#### 대응 전략

| 영역 | 전략 |
|------|------|

| 성장 단계 | 사업자등록 후 기업 인증키 전환. 공시검색이 무제한으로 전환되어 폴링 부담 완전 해소 |

| 높음 | AI 해석 오류 → 유저 손실 | 면책 문구, 신고 버튼 즉시 처리, 확신도 낮으면 "판단 보류" 표시 |
| 높음 | 규제 리스크 (자본시장법) | 투자자문업 등록 검토, "투자 권유 아님" 명시, 법률 자문으로 경계 설계 |
| 중간 | 대형 플랫폼 유사 기능 출시 | 빠른 PMF로 브랜드 선점, 공시 특화 정확도·깊이 차별화, B2B 채널 확보 |
| 중간 | LLM API 비용 급등 | 경량 모델 자체 구축 준비, 처리 비용 최적화, 복수 API 공급사 다변화 |
| 낮음 | DART API 정책 변경 | RSS·크롤링 병행 백업 경로 유지, API 의존도 분산 |
| 낮음 | 낮은 지불 의사 | 무료로 습관 형성 후 전환, "손실 방어" 프레이밍, 연간 결제 할인 |

## 16. 실행 로드맵

> 본 로드맵은 BM/GTM 관점의 사업 단계 계획이며, 단계별 진입 시점은 진행 상황에 따라 조정될 수 있다.

### Phase 1: MVP 구축
- DART OpenAPI 수집 파이프라인 구축
- 핵심 공시 유형 프롬프트 엔지니어링
- 카카오 알림톡 비즈 검수 신청
- 베타 테스터 100명 확보
- 법률 검토 (자본시장법)
- 최소 기능 웹 (종목 등록 + 알림 수신)
- 피드백 루프 (틀린 해석 신고)

### Phase 2: PMF 달성
- Pro 플랜 가격 A/B 테스트
- 정확도 개선 (파인튜닝 또는 RAG 강화)
- 리텐션 인터뷰 (30일 이탈 유저)
- 레퍼럴 프로그램 도입
- 모바일 앱 정식 출시
- 콘텐츠 마케팅 시작

### Phase 3: 성장
- Premium 플랜 출시 (섹터 분석 + 포트폴리오 리스크)
- 증권사 제휴 영업 (PoC 제안서)
- 인프라 스케일업 (Queue 기반 비동기, 오토스케일링)
- 초기 투자 유치 준비

### Phase 4: 확장
- 글로벌 공시 (미국 SEC EDGAR, 일본 EDINET)
- B2B API 상품화 및 화이트라벨
- AI 에이전트 ("이 공시에 어떻게 대응할까?")
- 집계 데이터 인사이트 상품
- Series A 준비
